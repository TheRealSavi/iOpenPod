# Internationalization

iOpenPod uses gettext catalogs for UI text. Source code keeps English strings
as the canonical msgids and wraps user-visible literals with `tr(...)` or the
`tr as _` alias.

## Workflow

1. Wrap new UI strings in `_(...)`.
2. Run `uv run python scripts/update_translations.py` to append missing msgids
   to `locale/<lang>/LC_MESSAGES/iopenpod.po`.
3. Fill in `msgstr` values in the `.po` file.
4. Run `uv run python scripts/compile_translations.py` to regenerate `.mo`
   files used at runtime.
5. Run `uv run python scripts/update_translations.py --check` before shipping
   to confirm catalogs are not behind source.

Do not add translation dictionaries to Python modules. Keep translations in
`locale/*/LC_MESSAGES/iopenpod.po`, and commit the compiled `.mo` files so the
installed wheel can load translations without requiring gettext tooling.

Changing language in Settings saves `AppSettings.language`, updates the process
gettext catalog, and rebuilds the window so existing widgets receive translated
labels.
